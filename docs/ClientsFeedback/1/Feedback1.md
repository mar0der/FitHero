# Product Feedback Summary: Fitness & Nutrition App

## Overview
This document summarizes critical feedback from Erika Szabo regarding current market gaps (specifically referencing limitations in Trainerize) and the planned technical roadmap for the new fitness platform. The goal is to create a highly customizable, time-efficient tool for personal trainers and their clients.

## 1. Hardware & Ecosystem Integration
The app should serve as a central hub for all health data to minimize manual entry.
* **Mandatory Integrations:** Apple Watch, Samsung, Garmin, Fitbit, Oura, and Whoop.
* **Requirement:** Seamless data flow of metrics like heart rate, sleep, and steps into the trainer dashboard.

## 2. Progress Tracking & Check-in UX
The "Check-in" process is the primary tool for demonstrating value and retention.
* **Side-by-Side Comparison:** A high-priority UI feature to display **previous week’s check-in data directly alongside the current week’s**. This allows trainers to visually demonstrate progress trends to clients instantly.
* **Customizable Documentation:** All forms—including PAR-Q, health history, injury questionnaires, and weekly check-ins—must be fully editable by the trainer rather than using fixed templates.

## 3. Training & Exercise Library
The library must be flexible enough to handle different terminology and equipment availability.
* **Controlled Exercise Swaps:** Trainers should be able to set "Pre-approved Alternatives" for movements. This allows clients to swap exercises when equipment is busy without choosing inappropriate substitutes.
* **Alias-Based Search:** The library must support multiple names for the same exercise (e.g., "Bench Press" and "Chest Press") to ensure trainers and clients can find movements regardless of their preferred terminology.
* **Client Emulation Mode:** Trainers need the ability to log in with a client profile to record demonstration videos and test the user experience within the native app environment.

## 4. Nutrition & Meal Planning
Move away from rigid, "cookie-cutter" templates that frustrate users.
* **Flexible Meal Structures:** The UI must support a dynamic number of meals (not restricted to a 3, 4, or 5-meal layout).
* **Customizable Caloric Weight:** Support for snacks or smaller meals that have independent caloric values, rather than forcing snacks to match the calories of main meals.

## 5. Intelligent Features & Automation (Phase 1)
Key technical features to be integrated into the initial design:
* **Strength Analytics:** Automated 1RM (One Rep Max) calculation based on initial strength inputs.
* **Heart Rate Intelligence:** Suggestions for HR zones tailored to the training plan and "Effort Ratings" generated by analyzing HR data post-workout.
* **Smart Suggestions:** Automated suggestions for alternate exercises.

## 6. Design Philosophy
* **Efficiency First:** The platform must solve the "complexity" problem of current apps. It should save the trainer time, not add hours of administrative work.
* **Trainer Control:** Ensure every variable is editable for the trainer to maintain professional oversight.
* **Simplicity for Clients:** While the trainer has deep control, the client-facing interface must remain clean and intuitive.
